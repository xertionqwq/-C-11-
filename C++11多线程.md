# 基于现代c++11的多线程



## 一.线程库的使用

+ **进程**： 运行中的程序
+ **线程**： 进程中的多个进程



### 1. 包含**头文件**

~~~c++
#include <thread>
~~~



### 2. **使用**线程

```c++
// 创建线程
std::thread t(functionName, args..);
if (t.joinable()) {
    t.join();// 主线程等待子线程结束
}
```

- **functionName** 为需要执行的**函数名**

- **args..** 为函数需要传递的**参数**(可包含多个)

  

- ***join()*** 函数是主线程**等待**子线程结束(需要包括, 否则报错)

- ***detach()*** 函数可以让子线程**脱离**主线程

- ***joinable()*** 函数判断 **t** 是否可以被join



### 3. 完整代码

```c++
#include<iostream>
#include<thread>
#include<string>   

// 进程: 运行中的程序
// 线程: 进程中的进程

void fun(std::string s) {
    for (int i = 0; i < 10; ++i) {
        std::cout << s << std::endl;
    }
}

void fun1() {
    for (int i = 0; i < 50; ++i) {
        std::cout << "fun1" << std::endl;
    }
}

int main() {
    std::thread t1(fun, "hello world");//主程序创建一个子线程t1，t1执行fun函数，主程序继续往下执行，不会等待
    bool isJoin = t1.joinable();//判断t1是否可以被join
    if (isJoin) {
        t1.join();//主程序等待t1执行完毕后，才继续往下执行
        // t1.detach();//主程序和t1分离，主程序继续往下执行，t1在后台继续执行
        // fun1();
    }
    fun1();
    std::cout << "return done" << std::endl;
    return 0;
}
```



























## 二. 线程函数的未定义错误

### 1. 函数参数为引用

+ 传入**临时变量**(错误)

+ **引用**变量要用***std::ref()***

~~~c++
void foo(int &x) {
    x += 1;
}
std::thread t(foo, 1);//传递临时变量,直接报错
// std::thread t(foo, a);
std::thread t(foo, std::ref(a));//使用std:ref()转为引用
~~~



### 2.**指针**或**引用**为**局部临时变量**或者已经**被释放**(非main函数)

```c++
传递指针指向局部变量
std::thread tt;
void test() {
    int a = 2;
    tt = std::thread(foo, std::ref(a));//a的地址已经清除
}

int main() { test(); tt.join(); }// 能过编译, 但是运行错误
```



### 3. 主程序指针意外释放

```c++
void func(int *p) {
    sleep(2);
    for (int i = 0; i < 10; i++){
        // 打印结果并非从91开始
        std::cout << *p << std::endl;
    }
}

int main() {
    //子程序还未运行完，主程序指针已被释放
    int *ptr = new int(91);
    std::thread mm(func, ptr);
    delete ptr;
    mm.join();
}
```



### 4. 类对象被意外释放

解决: 使用**智能指针**

```c++
class A{
public:
    void foo() { std::cout << "A::foo()" << std::endl;}
};

int main() {
	A a;
    std::thread t(&A::foo, &a);
    // 此处a因为某些原因导致提前释放
    t.join();
}
```



### 5. 线程入口为类的私有

解决: 使用**友元**

```c++
class A{
private:
    void foo() { std::cout << "A::foo()" << std::endl;}
};

std::shared_ptr<A> aa = std::make_shared<A>();
std::thread nn(&A::Afun, aa);// 无法访问到函数
nn.join();
```





















## 🔐三. std::mutex(互斥锁)

​	**核心作用**: 避免出现**数据竞争**

​	*数据竞争*: **多个线程**同时操作**同一块内存**, 可能在某时使其中一个线程对数据**操作失败**, 导致达不到想要的结果



### 1. 包含头文件

```c++
#include <mutex>
```



### 2. 使用mutex互斥锁

#### 	1.**函数lock()和unlock()**加解锁

​	作用: 使**加锁段**仅有**一个线程**可以进行操作, 其他线程阻塞

```c++
std::mutex mtx;
void add(int &a)
{
    mtx.lock();//锁之间的片段只有一个线程进行访问
    for (int i = 0; i < 2000000; i++) { 
        a++; 
    }
    mtx.unlock();//频繁加解锁会增加耗时
}

int main() {
    int a = 0;
    std::thread t2(add, std::ref(a));
    std::thread t4(add, std::ref(a));//t2,t4出现数据竞争
    t2.join();
    t4.join();
    std::cout << "a: " << a << std::endl;//如果不上锁，结果并非预期

    std::cout << "return done" << std::endl;
    return 0;
}
```



#### 	2. 死锁

​	**经典场景**: 两个线程t1，t2

​			t1先获取a, 再获取b
​			t2先获取b，再获取a
​			由于a，b同时上锁，程序锁死(互相等待)



​	**产生条件**: **互斥**：锁只能被一个线程持有（`std::mutex` 本身就是互斥的）

​			 **持有并等待**：线程持有一个锁，同时等待另一个锁（t1 持有 m1 等 m2，t2 持有 m2 等 m1）

​			 **不可剥夺**：锁不能被强制夺走，只能由持有者释放

​			 **循环等待**：线程间形成等待循环（t1→m2→t2→m1→t1）



​	**解决方式**: 两个线程**同步**获取互斥锁的顺序

````c++
std::mutex m1, m2;
void func1()
{
    m1.lock();// t1先拿到m1
    sleep(3);
    m2.lock();// t1想拿m2加锁, 但m2现在在t2手上
    m1.unlock();
    m2.unlock();
}
void func2() {
    m2.lock();// 由于t1在sleep, t2也拿到了m2
    m1.lock();// t2想拿m1加锁, 但m1现在在t1手上, 程序死锁
    m2.unlock();
    m1.unlock();
}

int main(){
    std::thread t1(func1);
    std::thread t2(func2);
    t1.join();
    t2.join();

    std::cout << "return done" << std::endl;
    return 0;
}
````



#### 	3. lock_guard()

​	作用: **自动加解锁**, 解决死锁问题

+ **构造**函数调用，互斥量自动**锁定**

+ **析构**函数调用，互斥量自动**解锁**

+ ***lock_guard***对象**无法移动和复制**，所以只在**局部作用域**中使用

  

  使用: 

  ```c++
  {
      std::lock_guard<std::mutex> lg(mtx);// 自动加锁
  }// 作用域结束, 自动解锁
  ```

  

#### 	4. unique_lock()

​	作用: 相对于lock_guard()有**更多函数**操作

```c++
std::unique_lock<std::mutex> lg1(mtx); // 自动加锁
std::unique_lock<std::mutex> lg2(mtx, std::defer_lock); // 需要用户自己加锁
```



+ 参数***std::defer_lock***让用户自己加解锁

+ 函数***try_lock()***会尝试加锁, 成功返回**true**,否则为**false**

+ 函数***try_lock_for()***尝试加锁, 直达**拿到锁**或者**时间耗尽**, 时间耗尽直接返回

+ 函数***try_lock_until()***在**规定时间**前尝试加锁, 到达时间未拿到锁, 直接返回

+ 对象***std::timed_mutex***支持时间操作

  

```c++
int shared_data = 1;
{	std::unique_lock<std::timed_mutex> lg(tmtx, std::defer_lock);
        // 如果等待2秒仍然拿不到，则直接跳过，防止死锁
        if (lg.try_lock_for(std::chrono::seconds(2))) {
            sleep(4);
            shared_data++;
        }
}
```

























## 四. std::call_once(单例模式)

+ **单例模式**: 保证一个类在整个程序生命周期内只有一个实例，并提供一个全局唯一的访问入口



### 1. 饿汉模式

​	利用**static**关键字

​	只允许构造**单个静态**对象

```c++
// 单例模式的日志
class Log {
private:
    // 无参构造设为私有
    Log() = default;
    static Log *log;
public:
    // 禁用拷贝构造和赋值构造
    Log(const Log &log) = delete;
    Log &operator=(const Log &log) = delete;

    static Log& getInstance() {
        static Log log;//饿汉模式
        return log;
    }
    void PrintLog(std::string message) {
        std::cout << __TIME__ << message << std::endl;
    }
};
```



### 2. 懒汉模式

​	利用**指针**

​	在需要时才构造

```c++
// 单例模式的日志
class Log {
    Log() = default;
    static Log *log;
public:
    Log(const Log &log) = delete;
    Log &operator=(const Log &log) = delete;

    static Log& getInstance() {
		static Log *log = nullptr; //懒汉模式
        if (!log) {
            log = new Log();
        }
        return *log;
    void PrintLog(std::string message) {
        std::cout << __TIME__ << message << std::endl;
    }
};
```



### 3. 使用call_once

+ 保证***func()***函数在多个线程中只被执行一次

```c++
template<class Callable, class... Args>
void call_once(std::once_flag &flag, Callable &func, Args&&... args);
```



```c++
static std::once_flag once;
// 单例模式的日志
class Log {
    Log() = default;
    static Log *log;
public:
    Log(const Log &log) = delete;
    Log &operator=(const Log &log) = delete;

    static Log& getInstance() {
        // call_once仅能在线程里面使用
        std::call_once(once, [](){
            if(!log) log = new Log; });
        return *log;
    }
    void PrintLog(std::string message) {
        std::cout << __TIME__ << message << std::endl;
    }
};
Log* Log::log = nullptr;
```

## 五. condition_variable(条件变量)



### 1. 生产者---消费者模型

![905DAB87D11FE1622ADB7C1B0612228D](C:\Users\xertion\Desktop\905DAB87D11FE1622ADB7C1B0612228D.jpg)



### 2. 使用condition_variable条件变量

```c++
#include <condition_variable>
```

​	**核心作用**: 用于**通知/等待**线程

+ **等待**: ***wait(lock, !isempty)*** 第一个参数为锁, 第二个为队列是否为空

+ **通知**: ***notify_one()*** 通知**一个线程**来取任务

  ​	  ***notify_all()*** 通知**所有线程**取任务



#### 完整代码

```c++
#include<iostream>
#include<unistd.h>
#include<thread>
#include<string>
#include<condition_variable>// 条件变量
#include<queue> // 任务队列
#include<mutex>

// 生产者和消费者模型
// 使用std::condition_variable进行通知
// notify_one()通知一个线程取任务
// notify_all()通知所有线程
std::queue<int> q_queue;
std::condition_variable g_cv;// 条件变量（用于线程通知/等待）
std::mutex mtx;              // 互斥锁（保护队列）
bool is_producer_done = false;

// 操作同一个对象q_queue,所有要加锁
// 派发任务
void Producer() {
    for (int i = 0; i < 10; i++) {
        {
            // 作用域：限制锁的持有时间，避免长时间占用锁
            std::unique_lock<std::mutex> lock(mtx);
            q_queue.push(i);
            // 通知一个线程来取任务
            g_cv.notify_one();
            std::cout << "Producer add: " << i << std::endl;
        }
        sleep(0.5);
    }
    is_producer_done = true;
}

// 完成任务
void Consumer() {
    while(true) {
        std::unique_lock<std::mutex> lock(mtx);
        // 如果队列为空，则需要等待
        // 第二个参数要用仿函数实现
        // 第二个参数为false时，线程阻塞在这里
        // 直到生产队列传来消息，继续消费
        g_cv.wait(lock, []
                  { return !q_queue.empty() || is_producer_done; });
        int value = q_queue.front();
        q_queue.pop();

        std::cout << "Consumer remove: " << value << std::endl;
    }
}

int main(){
    std::thread t1(Producer);
    std::thread t2(Consumer);
    std::thread t3(Consumer);
    t1.join();
    t2.join();
    t3.join();

    std::cout << "return done" << std::endl;
    return 0;
}
```

### 完整代码实现(大的来了)

```c++
#include<iostream>
#include<thread>
#include<unistd.h>
#include<chrono>
#include<condition_variable>
#include<functional>
#include<vector>
#include<queue>
#include<mutex>
#include<cstdio>

class ThreadPool{
    std::vector<std::thread> threads;   //线程池
    std::queue<std::function<void()>> tasks;//任务队列

    std::mutex mtx;//互斥锁
    std::condition_variable condition;//条件变量

    bool stop;      //线程池停止变量

public:
    ThreadPool(int numThreads) : stop(false) {
        for (int i = 0; i < numThreads; ++i) {
            // 消费者处理线程
            // 使用匿名函数,捕获this指针
            threads.emplace_back([this]{
                while(1) {
                    // 拿到任务的整个过程要上锁
                    std::unique_lock<std::mutex> lock(mtx);
                    // 条件变量，当任务队列为空，或者不再发出任务时阻塞
                    condition.wait(lock, [this]
                                   { return !tasks.empty() || stop; });
                    
                    // 线程已终止且不再有任务
                    if (stop && tasks.empty())
                        return;

                    //从任务队列中取出任务
                    std::function<void()> task{std::move(tasks.front())};
                    tasks.pop();//该任务离开任务队列
                    lock.unlock();//解锁，方便其他线程获取任务
                    task(); // 进行任务
                } }
            );
        }
    }
    ~ThreadPool() {
        {
            // 限定作用域，结束后自动解锁
            std::unique_lock<std::mutex> lock(mtx);
            stop = true;
        }
        //通知所有线程结束工作
        condition.notify_all();
        for (auto &thread : threads) {
            if (thread.joinable()) {
                thread.join();
            }
        }// 等待线程结束工作
         // 所有任务完成，销毁线程池
    }

    template<typename F, typename ... Args> //不确定参数有多少个，使用模板
    //&&万能引用
    void enqueue(F &&f, Args&&... args) {
        //std::bind()使用函数绑定器
        std::function<void()> task = std::bind(
            std::forward<F>(f),          // 转发函数对象（必须指定模板参数）
            std::forward<Args>(args)...  // 转发所有参数（逐个指定模板参数）
        );//由于使用了万能引用，所以要用完美转发保留表达式类型

        // 对任务队列开始操作，上锁
        {
            std::unique_lock<std::mutex> lock(mtx);
            tasks.emplace(std::move(task));//move()触发移动构造，防止深拷贝
        }
        condition.notify_one();//通知线程来取任务
    }
};

int main() {
    ThreadPool pool(5);//开辟5个线程的线程池

    for (int i = 0; i < 10; ++i) {
        pool.enqueue([i]()
                     {  printf("task : %d is running\n", i);
                        std::this_thread::sleep_for(std::chrono::seconds(1));
                        printf("task : %d is done\n", i); }
                    );
    }
    for (int i = 0; i < 10; ++i) {
        pool.enqueue([i]()
                     {  printf("task : %d is running\n", i);
                        std::this_thread::sleep_for(std::chrono::seconds(1));
                        printf("task : %d is done\n", i); }
                    ); // 用printf()避免线程争夺io资源
    }

    std::this_thread::sleep_for(std::chrono::seconds(2));
    std::cout << "return done" << std::endl;
    return 0;
}
```

## 八. std::atomic(原子操作)

+ **std::atomic**是c++11中的标准模板类, 用于多线程下的原子操作

  提供线程安全下的**数据共享**和**访问**

  避免*数据竞争*



### 1. 使用std::atomic

```c++
#include <atomic> // 包含头文件
std::atomic<int> shared_data(0); // 原子变量，保证线程安全
void fun() {
    for (int i = 0; i < 3000000; ++i) {
        // 效率更高的原子操作，避免了锁的开销
        shared_data++;
    }
}
```



### 2. 常用操作

+ ***load()***: 将变量的值加载至当前线程的缓存, 并返回这个值
+ ***store(val)***: 将**val**存储至原子变量中, 保证操作是原子性的
+ ***exchange(val)*** : 将**val**存储至原子变量中, 并**返回原来的值**
